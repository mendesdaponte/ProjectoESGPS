Consulta ( Cenário Principal - Doctor )

Pré Condição > O Doutor é um utilizador válido no sistema

Descrição > 1. O caso de utilização começa quando o doutor selecciona a opção pesquisar , onde vai surgir
uma janela com as horas da consulta ao longo do dia.
	    2. De seguida o Doutor selecciona a hora da consulta que pretende.
	    3. Abre-se uma janela onde vai se poder registar os sintomas e diagnostico de um determinado 
paciente, e se necessário passar medicação.
	    4. O Doutor confirma tudo o que registou.

Pós Condição > Os registos seram guardados nessa data de consulta

Registar/Procurar Paciente ( Cenário Principal - Search )

Pré Condição > (o/a) Atendente/Nurse/Doctor é um utilizador válido no sistema

Descrição  > 1. O caso de utilização começa quando o atendente  selecciona a opção pesquisar , onde vai aparecer uma
lista de pacientes e clica-se sobre o paciente pretendido.
	     2. De seguida abre uma janela com os dados do Paciente. 
	     3. Se pretender alterar/apagar os dados de Registo do Paciente , altera-se/apaga-se os dados.
	     4. Clica-se no "Update Patient" para os novos dados ficarem registados.

Pós Condição > A lista dos Pacientes é atualizada

Login System ( Cenário Principal - System Admin )

Pré Condição > O utilizador tem de estar registado no sistema

Descrição > 1. O "System Admin" inicia o software.
	    2. O "System Admin" faz Login.  
	    2. O "System Admin" cria com os campos nome e password para os restantes usuarios do programa.
	    3. O "System Admin" define  o acesso que cada usuario vai ter.
	    4. O Sistema valida os campos, fecha  "Login" e abre a janela a que tem acesso. 

Login System ( Cenário Alternativo - System Admin )
 

Descrição > 1. O "System Admin" inicia o software.
	    2. O "System Admin" faz Login. 
		a) O usuario introduz campos inválidos. 
		b)O sistema mostra uma mensagem de erro "Os dados que introduziu são inválidos" e retorna ao
		 passo 2 do Cenário Principal.
	    2. O "System Admin" cria os restantes usuarios do programa com os diversos dados pedidos.
	    3. O "System Admin" define  o acesso que cada usuario vai ter.
	    4. O Sistema valida os campos, fecha  "Login" e abre a janela a que tem acesso. 


Pós Condição:O usuario fica com acesso à sua área restrita do sistema
